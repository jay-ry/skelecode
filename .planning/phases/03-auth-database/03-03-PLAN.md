<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> middleware.ts | Unauthenticated request crosses; middleware enforces auth before route handler runs |
| middleware.ts -> protected pages | Only authenticated requests reach `/dashboard`, `/sprints`, `/skeleton` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-S-01 | Spoofing | `auth.protect()` call | mitigate | `await auth.protect()` — the `await` is enforced in acceptance criteria. Without await, Clerk 7 returns a Promise and the guard is a no-op (Pitfall 2). |
| T-03-E-01 | Elevation of privilege | middleware matcher | mitigate | `createRouteMatcher(["/dashboard(.*)", "/sprints(.*)", "/skeleton(.*)"])` is exhaustive for protected routes. Landing page `/` stays public per PROJECT.md landing-page requirement. Grep verifies matcher is exactly the locked list. |
| T-03-R-01 | Repudiation | Sign-in flow | transfer | Clerk handles audit logs, session rotation, and MFA. Out-of-scope to hand-roll. |
| T-03-I-04 | Information disclosure | `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | accept | Publishable key is designed to be client-safe. Secret key (`CLERK_SECRET_KEY`) stays server-only — no `NEXT_PUBLIC_` prefix in `.env.example` (Plan 01). |
</threat_model>
