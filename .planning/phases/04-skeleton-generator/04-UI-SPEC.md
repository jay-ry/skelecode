## Typography

All type is Tailwind utility-based — no CSS classes beyond what the existing codebase uses.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-relaxed`) | Rajdhani | Panel content, list items, descriptive text |
| Label | 12px (`text-xs`) | 600 (semibold) | 1 | Rajdhani | Section headers (FOLDER STRUCTURE, SPRINT 1 WIREFRAME) |
| Heading | 14px (`text-sm`) | 600 (semibold) | 1.2 | Rajdhani | Button labels, nav links, CTA text |
| Mono | 14px (`text-sm font-mono`) | 400 (regular) | 1.5 | Share Tech Mono | ASCII tree in FolderTree `<pre>`, monospace labels |

**Note on typographic hierarchy:** Body (14px regular) and Mono (14px regular) are the same numerical size but differ in font family (Rajdhani vs Share Tech Mono), which is sufficient to visually distinguish them. Label (12px semibold) is the only smaller size and is reserved for uppercase section headers only. No display-size headings (28px+) are used — this is a tool page, not a landing page.

**Rules:**
- Section labels (FOLDER STRUCTURE, SPRINT 1 WIREFRAME) use `text-xs font-semibold uppercase tracking-wide text-[#00ffe0] font-mono` — matches the `Section` component pattern in SprintCard.tsx
- Button text uses `text-sm` — matches `btnClass` in Header.tsx
- ASCII tree `<pre>` uses `font-mono text-sm text-[#c8f0ea] leading-relaxed` — monospace, full width, no wrapping (`whitespace-pre overflow-x-auto`)

## Color

All values are drawn from the established CSS custom properties in `globals.css`. No new colors are introduced.

| Role | CSS Variable | Hex | Usage |
|------|-------------|-----|-------|
| Dominant (60%) | `--background` | `#020408` | Page background, panel backgrounds, `<pre>` background |
| Secondary (30%) | `--bg-secondary` | `#050d14` | Button hover state, panel borders, card inset areas |
| Accent (10%) | `--accent` | `#00ffe0` | Reserved — see accent list below |
| Destructive | `--accent2` | `#ff003c` | Error messages only |
| Muted text | `--text-muted` | `#7abfb8` | Descriptive body text, empty state body copy |
| Foreground text | `--text` | `#c8f0ea` | Primary text, list items, tree lines |
| Border | `--border` | `rgba(0,255,224,0.15)` | All panel borders, button borders, iframe border |
| Border subtle | `--border-subtle` | `rgba(0,255,224,0.08)` | Dividers within panels, section separators |

**Accent (`#00ffe0`) is reserved for:**
- Section label text (FOLDER STRUCTURE, SPRINT 1 WIREFRAME)
- Logo text in Header
- "Copy to clipboard" and "Open in new tab" button border-hover state (`hover:border-[#00ffe0]`)
- "Saved ✓" SaveStatus indicator text
- Forward navigation link (→ Sprint 1 Wireframe or similar future nav)
- Streaming/loading indicator (if used)

**Accent is NOT used for:**
- Body text
- Tree line characters
- General button fill/background

**Destructive (`#ff003c`) is reserved for:**
- Error state messages only (generation failure, save failure)
- No destructive actions exist in this phase — no delete/confirmation dialogs

```
┌──────────────────────────────────────────────────────────────┐
│  Header (shrink-0, border-b border-[--border])               │

│                           │  received                        │
│  [Copy to clipboard]      │  [Open in new tab]               │
└───────────────────────────┴──────────────────────────────────┘
```

- Root: `flex flex-col h-screen bg-[#020408]`
- Below header: `flex flex-1 overflow-hidden` (two panels side-by-side)
- Each panel: `flex-1 flex flex-col overflow-y-auto px-4 py-4 border-r border-[rgba(0,255,224,0.15)]` (right panel omits `border-r`)
- Panel label: `text-xs font-semibold uppercase tracking-wide text-[#00ffe0] font-mono mb-3`
- `Generate Skeleton` button: centered above the two panels, visible only when not generating and no output yet — `flex justify-center py-6`

**Focal point:** The `Generate Skeleton` button is the primary visual anchor on page load. All other elements (panel labels, empty panels, header links) are visually subordinate to it. Once generation is complete, the focal point shifts to the left panel's ASCII tree, which is the primary deliverable of this page.

**Header implementation note:** Match the Header padding exactly from `Header.tsx` — `px-5 py-2.5`. This is an existing component constraint, not a spacing token. Do not alter the Header component's padding in this phase.

## Component Contracts

### FolderTree.tsx

- Container: `relative w-full`
- `<pre>`: `font-mono text-sm text-[#c8f0ea] leading-relaxed whitespace-pre overflow-x-auto bg-[#020408] p-0`
- "Copy to clipboard" button: positioned top-right of the tree panel, using `btnClass` pattern from Header.tsx — `text-sm px-3 py-1 border border-[rgba(0,255,224,0.15)] rounded text-[#c8f0ea] hover:bg-[#050d14] hover:border-[#00ffe0] transition-colors`
- On copy success: button label switches to "Copied!" for 1500ms then reverts to "Copy" — no toast library needed; local `useState` on the button
- Collapse/expand: click a directory line to toggle its children. Available post-streaming only (disabled while `isGenerating === true`). Collapsed directories show a `▶` prefix; expanded show `▼`. Text color for toggle indicator: `text-[#00ffe0]`
- Streaming state: lines append as they arrive; no animation; cursor is not shown

### WireframePreview.tsx

- Container: `w-full flex flex-col gap-2`
- Loading placeholder (before wireframe event received): `flex items-center justify-center h-[500px] border border-[rgba(0,255,224,0.15)] rounded text-sm text-[#7abfb8]` — text: "Generating wireframe..."
- iframe: `w-full h-[500px] border border-[rgba(0,255,224,0.15)] rounded` with `sandbox="allow-scripts"` — no `allow-same-origin`
- "Open in new tab" button: `btnClass` pattern — `text-sm px-3 py-1 border border-[rgba(0,255,224,0.15)] rounded text-[#c8f0ea] hover:bg-[#050d14] hover:border-[#00ffe0] transition-colors`
- Button row: `flex justify-end mt-1`

### Generate Button (in skeleton/page.tsx)

- Style: `text-sm px-4 py-2 border border-[rgba(0,255,224,0.15)] rounded text-[#c8f0ea] hover:bg-[#050d14] hover:border-[#00ffe0] transition-colors` (identical to sprints page pattern)
- Disabled state: `disabled:opacity-40 disabled:cursor-not-allowed`
- While generating: button label changes to "Generating..." and is disabled
- When done (`isDone=true`) and output exists: button label is "Regenerate" — same style, enabled

### SaveStatus Indicator

- Shown near the Generate button, appears after `[DONE]` + successful PUT
- Text: `text-sm text-[#00ffe0]` — "Saved ✓"
- Duration: persists until next generation cycle begins
- On save failure: `text-sm text-[#ff003c]` — "Save failed — try regenerating"
- No toast library — inline text indicator only (as specified in CONTEXT.md Claude's Discretion)

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Initial (no output) | Page load, no saved skeleton | Both panels empty; "Generate Skeleton" button centered below header |
| Restored (saved skeleton) | Page mount + GET returns data | Both panels populated; "Regenerate" button; no generate prompt text |
| Generating | Button clicked | Button → "Generating..." disabled; tree lines stream left; right panel shows "Generating wireframe..." |
| Done | `[DONE]` event | Button → "Regenerate" enabled; "Saved ✓" appears; Download enabled |
| Error (generation) | `event.type === 'error'` | Error message in teal-muted area below panels; "Try again" button |
| Error (save) | PUT fails | "Save failed — try regenerating" inline next to SaveStatus position |
| Empty guard | `sprints` is empty in context | Full-panel empty state (see Copywriting) |
| Loading (mount) | GET /skeleton in flight | Both panels show subtle loading indicator: `text-[#7abfb8] text-sm` — "Loading..." |

## Accessibility Contract

- `<pre>` element: add `aria-label="Folder structure tree"` and `role="region"` on its container div
- `<iframe>`: add `title="Sprint 1 wireframe preview"`
- Generate/Regenerate button: `aria-busy={isGenerating}` while generating
- Copy button: `aria-live="polite"` on the label span so "Copied!" is announced to screen readers
- Error messages: `role="alert"` so they are announced immediately
- Empty state link: descriptive link text "Plan your sprints →" (not "click here")
- All buttons must have visible focus ring — use Tailwind `focus:outline-none focus:ring-1 focus:ring-[#00ffe0]`
