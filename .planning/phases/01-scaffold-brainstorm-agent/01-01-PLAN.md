<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /api/copilotkit | Unauthenticated POST; accepts arbitrary chat content |
| /api/copilotkit → Anthropic API | ANTHROPIC_API_KEY leaves the server on every request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Information Disclosure | `frontend/.env.local` | mitigate | ANTHROPIC_API_KEY must never be committed. `.gitignore` already lists `.env.local` via create-next-app. Task 1 verifies `.gitignore` contains `.env.local` before completing. Key is never referenced as `NEXT_PUBLIC_*` (which would expose it in the browser bundle). |
| T-01-01-02 | Tampering | `/api/copilotkit` chat input | accept | No auth in Phase 1 by design. User input is passed to Claude as conversation data, not inserted into system prompts. CopilotKit runtime handles the system prompt separately via the `instructions` prop set in BrainstormChat (Plan 01-03). Input is not used for dynamic system prompt construction in this plan. |
| T-01-01-03 | Denial of Service | CopilotKit runtime — unbounded API calls | accept | Phase 1 has no rate limiting by design. The reviewer loop cap (Plan 01-02) limits backend calls per generation. CopilotKit's own chat protocol does not have per-session limits at this phase; rate limiting is deferred to Phase 5 (monetization). |
</threat_model>
